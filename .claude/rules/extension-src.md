# Extension Host Rules

- Import from `vscode` with named imports (`import { window, commands } from "vscode"`); the module is provided by the host at runtime and marked `--external:vscode` in esbuild.
- Register every disposable (commands, providers, listeners) into `context.subscriptions` in `activate()`.
- New commands: add to `contributes.commands` (and `menus.commandPalette` with a `when` clause) in `package.json`, implement in `src/commands/`, register in `src/extension.ts`.
- Webview HTML is generated by `src/utils/get-webview-content.ts`; it rewrites asset URIs with `webview.asWebviewUri` and selects the React page via `data-page`. Do not hand-write webview HTML in providers.
- Unit tests run in Node (not the extension host). The `vscode` module resolves to `tests/__mocks__/vscode.ts` via the Vitest alias — extend that mock when your code touches new vscode APIs.
- Type `onDidReceiveMessage` payloads explicitly (e.g. `{ command: string; text: string }`); untyped `any` payloads fail the type-aware Biome rules.
